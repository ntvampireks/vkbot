# VK Bot — Диалоговый бот для сообщества VK

Простой диалоговый бот для сообщества VK, использующий LongPoll API для получения сообщений в реальном времени.

## Возможности

- Обработка сообщений от пользователей в реальном времени через VK LongPoll API
- Поддержка диалогового режима с сохранением контекста и истории (последние 10 сообщений)
- Простая система намерений (intent classification) по ключевым словам
- SQLite хранение с автоматическим управлением сессиями
- Расширяемая архитектура обработчиков с автоматической регистрацией
- Rate limiting для защиты от превышения лимитов VK API (~3 сообщения/сек)
- Фильтрация собственных сообщений бота и проверка упоминаний

## Быстрый старт

### 1. Установка зависимостей

```bash
pip install -r requirements.txt
```

### 2. Настройка токена сообщества VK

#### Шаг 1: Создать приложение VK

1. Зайдите в [раздел разработчиков VK](https://dev.vk.com/)
2. Нажмите «Создать приложение»
3. Заполните:
   - **Название**: любое (например, «Мой бот»)
   - **Платформы**: выберите «Сообщества»

#### Шаг 2: Подключить API сообщества

1. Зайдите в [панель управления сообществом](https://vk.com/groups)
2. Выберите ваше сообщество
3. Перейдите в **Управление → Настройки → Работу с API**

#### Шаг 3: Создать токен доступа

1. В разделе **Работа с API** нажмите «Создать токен»
2. Выберите необходимые права:
   - `messages` — доступ к сообщениям (обязательно)
3. Скопируйте сгенерированный токен

#### Шаг 4: Включить LongPoll

1. В том же разделе **Работа с API** найдите **Long Poll API**
2. Нажмите «Подключить»
3. Запомните `server`, `key`, `v` — они понадобятся для ручного тестирования

### 3. Настройка переменных окружения

Создайте файл `.env` в корне проекта:

```bash
cp .env.example .env
```

Заполните переменные:

```env
# Токен сообщества (из шага 3)
VK_GROUP_TOKEN=your_token_here

# ID сообщества (без минуса, например 12345678)
VK_GROUP_ID=12345678

# Имя бота для проверки упоминаний (обязательно для работы)
VK_BOT_NAME=Бот

# Опционально: настройки сервера
BOT_HOST=localhost
BOT_PORT=8000

# Опционально: таймаут сессии диалога (часы)
DIALOG_TIMEOUT_HOURS=24
```

**Как узнать ID сообщества:**

- Зайдите в управление сообществом
- URL будет вида `https://vk.com/edit?act=manage&group=12345678`
- Число после `group=` — это ID

Или в разделе **Настройки** → **Адрес страницы** — это число в конце внутреннего адреса.

### 4. Запуск бота

```bash
python -m bot.main
```

Бот начнёт слушать сообщения и отвечать на них.

## Архитектура

```
VK LongPoll → EventHandler → on_message() → IntentClassifier
                                                    ↓
                                               Router → Handler
                                                    ↓
                                            MessageService → Ответ
                                                    ↓
                                            DialogService (сохранение)
```

### Компоненты

| Компонент | Описание |
|-----------|----------|
| `VKClient` | Клиент VK API с LongPoll, отправляет сообщения через `messages.send` |
| `EventHandler` | Обработка входящих событий, преобразует в `Message` объекты |
| `DialogService` | Управление контекстом диалогов, сохраняет в SQLite |
| `IntentClassifier` | Классификация намерений по ключевым словам, проверка упоминаний |
| `MessageService` | Отправка сообщений с rate limiting (3 сообщения/сек) |
| `Handlers` | Обработчики для разных типов диалогов (автоматическая регистрация) |

### Поток обработки сообщения

1. **VKClient.run_forever()** — слушает LongPoll API
2. **EventHandler.handle_event()** — преобразует VK события в `Message` объекты
3. **on_message()** — проверяет упоминание бота и наличие вложений
4. **IntentClassifier.classify()** — определяет тип запроса по ключевым словам
5. **Router** — выбирает обработчик по intent
6. **Handler.handle()** — генерирует ответ
7. **MessageService.send()** — отправляет ответ через VK API

## Добавление нового обработчика

1. Создайте файл в `bot/handlers/`:

```python
# bot/handlers/custom_handler.py
from bot.domains.message import Message
from bot.domains.dialog import Dialog
from bot.handlers.base_handler import BaseHandler


class CustomHandler(BaseHandler):
    @property
    def intent(self) -> str:
        return 'custom'

    def handle(self, message: Message, dialog: Dialog | None) -> str:
        return 'Ответ вашего обработчика'
```

2. Зарегистрируйте в `bot/main.py`:

```python
from bot.handlers import CustomHandler

def create_router():
    return {
        'greeting': GreetingHandler(),
        'help': HelpHandler(),
        'custom': CustomHandler(),  # ← добавьте
        'unknown': DefaultHandler(),
    }
```

3. Добавьте ключевые слова в `IntentClassifier`:

```python
# bot/services/intent_classifier.py
self.intents = {
    # ...
    'custom': ['ключевое слово 1', 'ключевое слово 2'],
}
```

## Работа с базой данных

Бот использует SQLite для хранения:

- `dialogs` — активные сессии диалогов
- `messages` — история сообщений (последние 10 на пользователя)

База создаётся автоматически при первом запуске в `storage/dialogs.db`.

### Пример SQL-запросов

```sql
-- Все активные диалоги
SELECT * FROM dialogs ORDER BY last_active DESC;

-- История сообщений пользователя
SELECT * FROM messages WHERE user_id = 12345 ORDER BY timestamp DESC;
```

## Часто задаваемые вопросы

### Как бот отличает свои сообщения от сообщений пользователей?

Бот фильтрует собственные сообщения, проверяя `out` флаг в событии VK LongPoll. Если `out=1`, сообщение отправлено ботом и игнорируется.

### Почему бот не отвечает на все сообщения?

Бот отвечает только на сообщения, где он упомянут (например, `@Бот привет`). Это предотвращает спам в общих чатах. Имя бота настраивается через `VK_BOT_NAME`.

### Как ограничить историю сообщений?

История автоматически ограничивается последними 10 сообщениями на пользователя. Это настраивается в [bot/domains/dialog.py](bot/domains/dialog.py).

### Что делать, если бот не получает сообщения?

1. Проверьте, что LongPoll подключён в настройках API
2. Убедитесь, что токен имеет права `messages`
3. Проверьте, что бот добавлен в «Служба поддержки» сообщества
